## 核心流程

### 1. 初始化流程

```
registerProvider() ──▶ Provider Semaphore 创建
                           │
registerApiKey() ────────▶ Key Semaphore 创建
                           │
                           ▼
                    Model Semaphore 创建 (每个模型)
```

### 2. 请求执行流程

```
call(options)
    │
    ▼
检查可用 Key
    │
    ▼
加入 PQueue (按优先级排序)
    │
    ▼
获取三级 Semaphore (Provider → Key → Model)
    │
    ▼
执行 PiAiAdapter.complete()
    │
    ▼
p-retry (重试逻辑)
    │
    ▼
pi-ai 底层调用
    │
    ▼
返回 LLMResponse { content, tokens, stopReason }
```

### 3. 流式请求流程

```
stream(options)
    │
    ▼
... (同上, 获取 Semaphore 等)
    │
    ▼
PiAiAdapter.streamWithRetry()
    │
    ▼
for await (event of pi-ai stream)
    │
    ▼
mapEvent() ──▶ StreamEvent
    │
    ▼
yield { type, delta, accumulatedContent, tokens }
```

## 数据流

### 请求数据流

```
User ──▶ CallOptions ──▶ RequestScheduler ──▶ PiAiAdapter ──▶ pi-ai ──▶ LLM API
```

### 响应数据流 (流式)

```
LLM API ──▶ pi-ai ──▶ AssistantMessageEvent ──▶ mapEvent() ──▶ StreamEvent ──▶ User
```

## 关键设计决策

1. **三级 Semaphore**: Provider → Key → Model 顺序获取,避免死锁
2. **原子性获取**: 同时获取三级许可,要么全拿,要么不拿
3. **优先级队列**: 使用 p-queue,支持 0-N 优先级
4. **重试策略**: 仅对 429/5xx/网络错误重试,业务错误不重试
5. **Token 统计**: 复用 pi-ai 的 usage 事件,不自己估算
